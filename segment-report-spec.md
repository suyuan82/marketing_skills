# 细分人群报告格式规范

## 概述

细分人群报告是**第一层交付物** — 策略级研究文档，将每个发现的用户分群以完整证据链呈现。可直接用作产品简报、策略提案或营销计划中的研究附录。

---

## 文档结构

```
报告标题
├── 研究概览
│   ├── 研究目标
│   ├── 数据范围（subreddit、时间、各阶段数量）
│   ├── 方法论说明
│   └── 分群总览（数量 + 名称一览表）
├── 分群 1
│   ├── 分群名称（双语）
│   ├── 分群描述
│   ├── 人口统计推断
│   ├── 行为与态度标签
│   └── 原声语录
├── 分群 2
│   └──（同上结构）
├── ...
├── 跨分群对比
└── 局限性说明
```

---

## 各部分规范

### 研究概览

简明摘要（300-500 字），覆盖：
- 研究对象和原因
- 使用了哪些 subreddit / 关键词
- 时间范围和漏斗各阶段数据量
- 方法论：简要说明采用了自下而上 / 扎根理论方法
- 一张表格列出所有分群名称及一句话摘要

---

### 分群章节

#### 分群名称

双语格式：**中文名（English Name）**

作为二级标题出现。

<example>
## [分群中文名]（[Segment English Name]）
</example>

#### 分群描述

200-300 字，中文。涵盖：
- 这些用户是谁（推断的身份和背景）
- 他们与产品/品类的关系有何特征
- 他们最关心什么
- 他们在线上讨论中的行为特点
- 与其他分群的区别

使用第三人称。保持分析性笔调——客观、有证据支撑、非推广性质。

领域专业术语：中文为主，首次出现时括号附英文。

<example>
这一分群由深度[活动类型]爱好者组成，他们将[产品品类]视为[核心活动]体验的重要组成部分，而非单纯的功能性消费。他们在选品上极为严苛，将耐用性（durability）和便携性（portability）置于[其他维度]之上……
</example>

#### 人口统计推断（Demographic Inference）

以结构化列表呈现。**每条推断必须附推理依据** — 数据中的什么证据支持这个判断。

必填字段：
- **年龄范围（Age Range）**
- **性别倾向（Gender Skew）** — 如可检测；否则注明"数据中无明显信号"
- **地域特征（Geographic Signals）**
- **收入/消费水平（Spending Level）** — 从产品提及、价格敏感度信号推断
- **生活阶段（Life Stage）**

<example>
- **年龄范围**：28-45 岁。依据：频繁提及"after 10 years of [activity]""with my kids"等表述，暗示有一定人生阅历和家庭责任的成年群体。
- **消费水平**：中高。依据：多次讨论 $[高价位段] 产品且不以价格作为首要否决因素，关注点集中在材质和耐久性上。
</example>

#### 行为与态度标签（Behavioral & Attitudinal Tags）

每个分群 6-12 个标签，以行内标签形式呈现，每个标签双语。

<example>
`深度研究者（Heavy Researcher）` `品牌忠诚度高（High Brand Loyalty）` `社区活跃贡献者（Active Community Contributor）` `[偏好维度]偏好（[Preference] Preference）` `价格脱敏（Price Insensitive）` `长期使用视角（Long-Term Usage Mindset）`
</example>

#### 原声语录（Verbatim Evidence）

每个分群 ≥15 条原声。这是最关键的部分 — 支撑整个分析的原始证据。

**每条语录格式**：

```
> [英文原文，完整复制 — 不纠正语法、不截断、不改写]
>
> [忠实的中文翻译]
>
> — 来源：r/[subreddit] · [大致日期] · ↑[点赞数] · 💬[评论数] · [原帖链接](URL)
```

**规则**：
- 英文原文必须原封不动 — 保留原始拼写、标点、俚语、表情符号
- 中文翻译忠实自然；俚语不过度本地化，但需确保可读性
- **原帖链接必须提供** — 在采集阶段同步记录，格式为 Markdown 链接 `[原帖链接](https://www.reddit.com/r/...)` ；如采集时确实无法获取，填写 `—`，但字段不可省略
- 按与该分群核心特征的相关性排序，而非按日期或点赞数
- 如精确元数据不可用，标注已知信息，未知项标记"—"

<example>
> I've had my [品牌A产品] for 3 years and it's been on every single [活动] trip. The thing is indestructible. Tried switching to [品牌B] but the plastic felt like it'd crack in my pack after a week.
>
> 我的[品牌A产品]已经用了 3 年，每次[活动]都带着。这东西简直坚不可摧。试过换成[品牌B]，但那塑料感觉放背包里一周就会裂。
>
> — 来源：r/[subreddit] · 2025-09 · ↑87 · 💬34 · [原帖链接](https://www.reddit.com/r/[subreddit]/comments/[post_id]/)
</example>

### 跨分群对比

一张对比表，突出所有分群在关键维度上的差异。

建议维度（根据数据中实际涌现的差异化维度调整）：

| 维度 | 分群 1 | 分群 2 | 分群 3 | … |
| --- | --- | --- | --- | --- |
| 核心诉求 | | | | |
| 价格敏感度 | | | | |
| 使用场景 | | | | |
| 认知水平 | | | | |
| 品牌关系 | | | | |
| 信息获取方式 | | | | |

具体维度应来源于编码过程中发现的、在分群间区分力最强的维度，而非预先指定。

---

### 局限性说明

必含部分。简要说明：
- 样本偏差：Reddit 用户群体在人口统计上的固有倾斜（英语为主、科技素养较高等）
- 社区选择偏差：纳入了哪些社区，可能缺失了哪些视角
- 时间窗口限制
- 证据量较低或边界模糊的分群（如有）
