# 自下而上分析方法论

## 方法论基础

本技能的分析方法受扎根理论（Grounded Theory）启发：细分人群结论必须**从数据中涌现**，而非事先设定。

在社交媒体用户研究的语境下，这意味着：

- **不**从预设的分群矩阵出发（例如"技术型 vs. 非技术型 × 高预算 vs. 低预算"）
- **不**在阅读数据前决定分群数量
- **要**逐条阅读帖子、标注观察到的内容，让分群自然形成

---

## 分步方法

### 第一步：开放编码（Open Coding）

逐条阅读筛选后语料（第二阶段 2c 的输出）。对每条帖子，分配一个或多个**编码** — 简短的描述性标签，捕捉用户正在表达的内容。

**规则**：
- 尽可能使用用户自身的语言（活体编码 / In Vivo Coding）
- 一条帖子可以有多个编码
- 此阶段不合并、不分组编码 — 保持颗粒度
- 典型的 60-100 条语料会产生 80-200 个初始编码

**观察维度**（作为观察视角，不是预设分类框）：

| 维度 | 关注什么 |
| --- | --- |
| 行为（Behavior） | 他们做什么？怎么使用产品？日常流程是什么？ |
| 动机（Motivation） | 他们为什么关心？什么驱动了兴趣或购买？ |
| 痛点（Pain Point） | 什么让他们沮丧？什么缺失或出了问题？ |
| 态度（Attitude） | 对品类、品牌或社区持什么态度？ |
| 场景（Context） | 在哪里、什么时候、和谁一起使用？ |
| 认知水平（Knowledge Level） | 新手、中级、专家？如何体现出来？ |
| 决策标准（Decision Criteria） | 选择或推荐时看重什么因素？ |

<example>
帖子原文："I've been using [品牌A] for 3 years camping and it still works perfectly. Tried [品牌B] but the plastic feels cheap and I worry about BPA. For $80 I expect better build quality."

分配编码：
- 户外露营场景（outdoor-use-camping）
- 长期产品忠诚（long-term-product-loyalty）
- 材料安全顾虑（material-safety-concern）
- 做工质量作为决策标准（build-quality-as-decision-criterion）
- 价格-品质预期（price-quality-expectation）
- 品牌比较行为（brand-comparison-behavior）
</example>

### 第二步：主轴编码（Axial Coding）

将相关编码归入更高层级的**类别**。识别类别之间的关系。

**流程**：
1. 列出所有编码
2. 寻找在同一帖子中频繁共现的编码
3. 识别因果关系、场景关系或条件关系
4. 形成候选**类别聚簇** — 这是分群的雏形

**核心提问**："哪些编码似乎在描述**同一类人**——在他们与这个产品/品类的关系上？"

<example>
聚簇 A（初步）："品质至上的户外深度玩家"
- 户外露营场景
- 长期产品忠诚
- 做工质量作为决策标准
- 材料安全顾虑
- 愿意支付溢价
- 装备评测行为

聚簇 B（初步）："效率导向的通勤族"
- 办公室日常使用
- 速度优先
- 紧凑体积需求
- 低维护偏好
- 价格敏感
- 咖啡知识较少
</example>

---

### 第三步：选择编码 / 分群结晶（Selective Coding / Segment Crystallization）

将候选聚簇精炼为独立的分群。

**验证标准 — 每个分群必须同时通过三项**：

1. **内部一致性（Internal Coherence）**：成员共享可识别的、有意义的行为-态度-需求模式
2. **外部区分度（External Distinction）**：与其他每个分群都有清晰差异——不仅仅是子变体
3. **证据充分性（Evidential Grounding）**：筛选后语料中有 ≥15 条原声可归属于此分群

**边缘情况处理**：
- 可归入多个分群的帖子：明确标注重叠，不强制归入单一分群
- 不属于任何分群的帖子：暂时搁置。如果 >15% 的语料未被分配，考虑是否遗漏了分群
- 两个高度重叠的候选分群：合并，寻找更高层级的命名

**经验参考**：常见结果在 3-6 个分群之间。少于 3 个可能意味着话题过窄或语料不足；多于 6 个可能存在过度拆分——检查是否有可合并的组。但最终数量完全由数据决定，不强制落入此范围。

---

### 第四步：分群命名

每个分群获得一个双语名称：简洁的中文名 + 英文对应。

**命名原则**：
- 名称应捕捉分群的**定义性特征** — 什么让他们成为"他们"
- 优先使用具体的行为或态度描述词，避免抽象标签
- 避免泛化命名——如果一个分群感觉"泛泛而谈"，分析可能需要进一步细化

<example>
好的命名：
- 极致户外玩家（Hardcore Outdoor Enthusiast）
- 效率通勤族（Efficiency-First Commuter）
- 精打细算的入门者（Budget-Conscious Newcomer）
- 社区意见领袖（Community Opinion Leader）

不好的命名：
- 用户群 A（User Group A）— 无意义
- 中端用户（Mid-Range User）— 仅基于价格的单一维度
- 年轻人（Young People）— 仅人口统计，无行为洞察
</example>

---

### 第五步：验证检查（Validation Review）

在产出最终交付物前，逐项检查：

| 检查项 | 通过标准 |
| --- | --- |
| **覆盖率** | ≥85% 的筛选后语料已被分配至某个分群 |
| **均衡性** | 没有单一分群包含 >50% 的帖子（除非数据确实如此倾斜，需明确说明） |
| **证据** | 每个分群有 ≥15 条可归属的原声语录 |
| **区分度** | 读者仅凭分群描述，就能正确地将一条新帖子归入正确分群 |
| **命名** | 每个名称通过"利益相关者能否立即理解这是什么样的人？"测试 |

如任一项未通过，返回第二至第四步迭代，再进入交付物产出。

## 分析诚信原则

- **警惕确认偏误（Confirmation Bias）**：分析者必须主动寻找**反驳**涌现模式的证据，而不仅仅寻找确认性证据
- **重视少数声音**：一个小但内部一致的聚簇（即使只有 5-8 条帖子）可能代表重要的新兴分群——不要仅因为数量少就丢弃
- **过程透明**：在最终报告中简要说明分析过程，注明局限性（如 subreddit 选择偏差、英语偏差、时间窗口限制）
